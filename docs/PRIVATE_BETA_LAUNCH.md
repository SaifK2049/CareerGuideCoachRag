# Masari private-beta launch runbook

Masari is configured for a free, invitation-only beta. Billing code remains dormant behind `billingEnabled: false`; do not add Stripe secrets or enable Premium buttons for this stage.

## Implemented beta controls

- Global public signup is disabled in the local Supabase model; hosted signup must also be disabled in Auth settings.
- Invited users record the current beta terms and privacy-notice version.
- All career, analysis, feedback, usage, and file rows are protected by user ownership policies.
- Successful analyses, source excerpts, model version, and timestamps are persisted.
- Analysis requests are idempotent. Failed provider calls release the reserved monthly quota.
- Beta limits are three paths, 20 job descriptions, 50 evidence items, and ten successful analyses per month.
- Browser-facing functions use an origin allowlist and per-user rate limits.
- Users can export account data, clear their workspace, and delete an account after password reauthentication.
- Billing and public signup controls are hidden in the browser.

## Owner-supplied launch values

These values cannot be generated by the application:

1. A production hostname, for example `https://app.example.com`.
2. An OpenAI API key with a project budget alert and spending cap.
3. A Cloudflare Turnstile production widget for the exact hostname.
4. A transactional SMTP account and a verified sending domain.
5. The beta operator's legal identity, privacy contact, and support contact.
6. A backup policy. Supabase Pro is recommended before storing real CVs.

The included `privacy.html` and `beta-terms.html` are beta drafts. Add the operator identity and obtain appropriate legal review before inviting external testers.

## Supabase configuration

In Authentication settings:

- disable new-user signup while leaving email/password login enabled;
- set the Site URL to the production hostname;
- allow only the production callback and explicitly trusted preview callbacks;
- configure custom SMTP and branded invite/reset emails;
- enable Cloudflare Turnstile using its secret key;
- keep email confirmation enabled where applicable;
- require a minimum ten-character password with upper-case, lower-case, and number characters;
- enable leaked-password protection if the project plan supports it;
- enable MFA for every Supabase organisation administrator.

Invite testers with the Supabase admin invitation flow. Do not re-enable the Create account interface.

## Edge Function secrets

Set only server-side secrets:

```bash
npx supabase secrets set OPENAI_API_KEY=...
npx supabase secrets set OPENAI_MODEL=gpt-5-mini
npx supabase secrets set APP_URL=https://app.example.com
npx supabase secrets set ALLOWED_ORIGINS=https://app.example.com
```

For multiple exact preview origins, provide a comma-separated `ALLOWED_ORIGINS` value. Never use `*`.

## Deploy Supabase

```bash
npx supabase link --project-ref YOUR_PROJECT_REF
npx supabase db push
npx supabase functions deploy analyze-career
npx supabase functions deploy export-account
npx supabase functions deploy delete-account
```

Checkout, portal, and webhook functions do not need to be redeployed for the free beta unless their shared security changes are required in the hosted project. Keep Stripe secrets unset.

## Deploy Cloudflare Pages

Use:

- build command: `npm run build`
- output directory: `dist`
- Node.js: 22

Required build variables:

```text
PUBLIC_SUPABASE_URL
PUBLIC_SUPABASE_PUBLISHABLE_KEY
PUBLIC_TURNSTILE_SITE_KEY
PUBLIC_BETA_MODE=true
PUBLIC_BILLING_ENABLED=false
PUBLIC_SIGNUP_ENABLED=false
PUBLIC_FEEDBACK_ENABLED=true
```

`PUBLIC_TERMS_URL` and `PUBLIC_PRIVACY_URL` are optional; the bundle defaults to the included pages.

## Acceptance gate

Complete this with two independent accounts before inviting testers:

- [ ] An uninvited visitor cannot create an account.
- [ ] An admin invitation, password setup/reset, sign-in, session refresh, and sign-out work from the production hostname.
- [ ] Turnstile is enforced for sign-in and password reset.
- [ ] The first user accepts the current beta terms and completes onboarding.
- [ ] A second user cannot read or mutate the first user's profile, jobs, evidence, analyses, feedback, usage, or CV PDF.
- [ ] A text-based PDF extracts; a scanned PDF shows the OCR-not-supported message.
- [ ] A live OpenAI analysis produces saved findings with working source excerpts.
- [ ] Retrying the same request does not consume a second quota unit.
- [ ] A forced OpenAI failure does not consume quota.
- [ ] Feedback arrives without CV or job text in its context.
- [ ] Admin analytics rejects non-admin accounts and excludes CV, job-description, evidence, interview, notes, contacts, and AI-output content.
- [ ] Product and operational telemetry contains only controlled fields and the 13-month retention job is active.
- [ ] Account export contains the profile, paths, jobs, evidence, analyses, audit events, feedback, and stored-file metadata.
- [ ] Workspace clear removes database content and stored CV files.
- [ ] Password-confirmed account deletion revokes the session and removes the account.
- [ ] Production responses contain CSP, HSTS, frame, content-type, and referrer headers.
- [ ] Security and performance advisors have no unresolved errors.
- [ ] A backup restore has been rehearsed.

Invite three testers first. Expand to ten, then 25, only if there are no privacy, deletion, or unrecoverable analysis issues.
